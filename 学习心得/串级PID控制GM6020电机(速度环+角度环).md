# 一、GM6020 电机串级 PID（角度 + 速度）闭环运行逻辑
串级 PID 核心是「角度外环定目标、速度内环保执行」，角度环输出作为速度环的目标值，速度环输出最终扭矩指令，整体逻辑如下：
* **初始化：**
CAN 配置（1Mbps 波特率、过滤电机 ID）+ 电机参数初始化（ID、角度限位、速度上限 / 下限、扭矩限幅）；
串级 PID 专属初始化：角度环 PID 参数（P/I/D）、速度环 PID 参数，双环寄存器清零；

  
* **指令预处理：**
设定目标角度（需在角度限位范围内），确认零位基准有效（避免角度偏移）；
对目标角度做平滑处理（如一阶低通滤波），防止阶跃指令导致电机冲击。
反馈采集与滤波：
实时接收电机 CAN 反馈的实际角度（主反馈）、实际转速（辅反馈）；
分别滤波：角度值做滑动平均滤波（消机械抖动），转速值做一阶滤波（防数据跳变），异常值直接丢弃。
串级 PID 核心计算：
外环（角度环）：计算角度偏差（目标角度 - 实际角度）→ 角度环 PID 运算 → 输出作为速度环的目标速度（限幅，不超速度上限）；
内环（速度环）：计算速度偏差（角度环输出的目标速度 - 实际转速）→ 速度环 PID 运算 → 输出扭矩 / 电流指令（限幅，不超电机额定扭矩）；
偏差保护：角度偏差超限（如＞设定阈值）→ 降低角度环 P 增益；速度偏差超限→ 临时限幅速度环输出，避免电机超速 / 冲击。
* **指令下发:**
将速度环输出的扭矩指令封装为 CAN 帧，≥200Hz 均匀下发（串级 PID 建议 200-500Hz，匹配双环响应特性）；
单帧发送失败仅重试该帧，重试期间沿用当前扭矩指令，不中断双环 PID 计算。
* **异常处理：**
角度异常（偏差超限 / 零位丢失）：暂停角度环 PID，速度环目标速度设 0，下发降速指令；
速度异常（偏差超限 / 超速）：限幅速度环输出，降低角度环输出的目标速度；
通信超时 / 电机故障：立即下发零扭矩指令，重置串级 PID 参数，电机停稳后重新校准零位；
正常停止：目标角度线性趋近当前实际角度，角度环输出逐步降 0，速度环跟随降 0，最终扭矩为 0。
# 二、串级 PID 的 CAN 库调用逻辑
串级 PID 对 CAN 通信的实时性、采样频率要求更高，调用逻辑需匹配双环协作时序，核心如下:

* **初始化:** CAN_Init→配置电机 ID 过滤→CAN_Start + 串级 PID 参数初始化 + 电机零位校准（仅上电 / 复位执行 1 次）；
控制周期（2-5ms）：CAN_Receive_Msg（读取实际角度 + 实际转速）→ 角度 / 转速分别滤波 → 角度环 PID 计算（输出目标速度）→ 速度环 PID 计算（输出扭矩）→ CAN_Send_Msg（下发扭矩指令）→ 检查发送状态。
时序 / 频率规则：
反馈接收：中断 / 轮询频率≥1000Hz（串级需高采样率，避免外环角度计算滞后）；
串级 PID 计算 + 指令下发：≥200Hz（2-5ms 周期），速度环计算频率需≥角度环（或同频），保证内环响应更快；
优先级：反馈接收＞速度环 PID 计算＞角度环 PID 计算＞指令下发＞日志记录（中断仅缓存反馈数据，不做 PID 计算）。
* **容错规则：**
单帧发送失败：立即重试（最多 3 次），重试期间沿用当前扭矩指令（避免串级 PID 输出突变导致角度 / 速度波动）；
接收超时（无角度 / 转速反馈）：暂停串级 PID 计算，下发上一周期扭矩指令，超时超 3 个周期则发零扭矩；
PID 输出超限：角度环输出强制限幅到速度阈值，速度环输出强制限幅到扭矩阈值，合规后再封装 CAN 指令。
